# MyFXBook Verification Runbook — публичный верифицированный трек

**Дата:** 2026-06-06
**Статус:** MVP-runbook (исполнять, когда пойдёт реальный/форвард live-счёт)
**Зачем:** третье-сторонняя верификация результатов — главный сигнал доверия для проспектов (kill-criterion #1, см. `docs/business/2026-06-06-unit-economics-honest.md`). Своя equity-кривая на своём дашборде проспекту не верится; MyFXBook трекает независимо.

> ⚠️ **Предусловие:** нужен живой счёт с НАКОПЛЕННЫМ треком (≥ нескольких недель сделок). До этого виджет нечем наполнять — см. `performance/` (внутренний go/no-go) и forward-track.
> ⚠️ **Юридическое:** до подключения реальных данных landing показывает ПЛЕЙСХОЛДЕРНЫЕ числа (помечены в `docs/business/landing.html`). НЕ публиковать landing с выдуманными числами — это недостоверная реклама (оферта 7.1.2). Сначала заменить реальными из MyFXBook.

---

## Шаги подключения

### 1. Аккаунт и подключение счёта
- [ ] Зарегистрировать аккаунт на [myfxbook.com](https://www.myfxbook.com).
- [ ] Add Account → **Automated tracking via Investor (read-only) password**. Ввести: сервер брокера, номер счёта, **investor-пароль** (НЕ торговый — read-only, MyFXBook не сможет торговать).
- [ ] Дождаться синхронизации истории (MyFXBook подтянет сделки сам).

### 2. Сделать публичным
- [ ] Settings портфеля → Privacy: **Public**. Выбрать, что показывать (обычно: equity, drawdown, trades; **скрыть** баланс в абсолюте, если не хочешь светить размер).
- [ ] Включить «**Verified**»-бейджи: MyFXBook верифицирует **Authenticity** (через investor-пароль — сделки реальны) и **Track Record** (по времени). Это и есть сигнал доверия.
- [ ] Скопировать публичный URL: `https://www.myfxbook.com/members/<user>/<portfolio>/<id>`.

### 3. Получить embed-виджет
- [ ] На странице портфеля → **Widgets** (или Tools → Widgets). Выбрать виджет (Equity chart / Stats / Custom). MyFXBook сгенерирует `<iframe>`-сниппет вида:
  `<iframe src="https://widgets.myfxbook.com/api/get-custom-widget.png?...&id=<WIDGET_ID>" ...></iframe>`
  (точный формат — из генератора MyFXBook на момент подключения; бывают png-виджеты и JS oneclick-виджеты).
- [ ] Записать `WIDGET_ID` / iframe-src и публичный URL портфеля.

### 4. Встроить в landing
- [ ] В `docs/business/landing.html`, секция `#track` (поиск `<!-- MYFXBOOK-INTEGRATION -->`):
  - заменить `href="MYFXBOOK_PORTFOLIO_URL"` на реальный публичный URL (шаг 2);
  - раскомментировать слот виджета и вставить iframe-сниппет (шаг 3), убрав рисованную SVG-заглушку;
  - заменить ПЛЕЙСХОЛДЕРНЫЕ числа в `track-stats` и `hero-stats` реальными из MyFXBook (или удалить, оставив виджет единственным источником).
- [ ] Снять все маркеры `<!-- PLACEHOLDER ... -->` после замены.

### 5. Поддержание
- [ ] MyFXBook обновляется автоматически (read-only sync). Захардкоженные на landing числа — кэш; либо периодически синхронизировать вручную, либо положиться на виджет (живой).
- [ ] При смене счёта/брокера — повторить шаги 1–4 с новым investor-паролем.

## Безопасность / нюансы
- **Только investor (read-only) пароль** — никогда не торговый. MyFXBook с ним не может торговать, только читать.
- **Демо vs реал:** MyFXBook трекает оба, но проспекты дисконтируют демо (оферта 7.1.3). Для доверия нужен **реальный** счёт (или явно помеченный форвард).
- **Размер счёта:** при желании скрыть абсолютные суммы (показывать только %/просадку) — приватность портфеля это позволяет.
- **Альтернатива:** FXBlue — аналогичный сервис; шаги похожи. MyFXBook популярнее в форекс-рознице.
